Part 2 — Account Rotation in queue-crawler.js

Changes to [scraper/queue-crawler.js](scraper/queue-crawler.js):

Before each scrape job:

const account = await accountPool.acquireAccount();
// inject into CONFIG: cookies, csrfToken, appId
CONFIG.cookies    = account.cookies;
CONFIG.csrfToken  = account.csrf_token;
CONFIG.igAppId    = account.app_id;

Rotation triggers (replace current VPN rotation):





429 received → markRateLimited(id, 2hr), re-queue job, pick next account



After daily_limit reached → account skipped until next day



3+ errors in 1 hour → account paused 4 hours



Per-job cooldown: release account for 10–15s after each scrape so other workers can use it

Rotation algorithm: weighted round-robin — prefer accounts with lowest daily_scrapes and oldest last_used.



Part 3 — Scale-Safe Delay Tuning

New delay profile in queue-crawler.js for 10+ accounts:

const DELAYS = {
  betweenJobs:       [6000, 10000],  // per worker — 10 accounts means 6s each = safe
  betweenPages:      [1500, 2500],
  afterSuggestions:  [2000, 3000],
  onRateLimit:       [7200000, 7200000], // 2hr hard pause for that account
  cooldownEvery:     40,             // jobs before cooldown
  cooldownMs:        [45000, 60000], // 45–60s pause
};

With 10 workers each waiting 6–10s, combined throughput = ~400–600 profile jobs/hour = ~4,000–5,000/day safely.

Part 5 — Profile Freshness & Refresh Loader

New field saved on every scrape (already in scraped_at):

{ "scraped_at": "2026-04-01T00:00:00Z", "refresh_priority": 1 }

refresh_priority values:





1 = standard (refresh every 14 days)



2 = high-value influencer > 100k followers (refresh every 7 days)



3 = active (refresh every 3 days, if posted in last 3 days)

New file: scraper/refreshScheduler.js

Runs as a cron job every hour:

// Query MongoDB for stale profiles
const stale = await db.collection('insta_Profiles').find({
  $or: [
    { refresh_priority: 1, scraped_at: { $lt: daysAgo(14) } },
    { refresh_priority: 2, scraped_at: { $lt: daysAgo(7)  } },
    { refresh_priority: 3, scraped_at: { $lt: daysAgo(3)  } },
  ]
}).limit(500).toArray();

// Re-enqueue them on the scrape queue
for (const p of stale) {
  await scrapeQueue.add('scrape', { username: p.data.username, isRefresh: true });
}





Capped at 500 refresh jobs per hour so they don't starve new discovery



isRefresh: true flag skips the follower-range filter (don't discard known profiles)

Change detection (lightweight)

Before doing a full re-scrape, do a cheap profile-only HTTP fetch (Method 1 only — no posts API):





If followers_count or posts_count changed → full re-scrape with posts



If unchanged → update scraped_at only, skip posts API (saves ~80% of API calls on stable profiles)
